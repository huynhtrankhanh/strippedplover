# Stripped Plover Manifesto
1. Plover is to run without external dependencies
2. Plover is to run without a UI
3. Plover is to run without capturing the keyboard
4. Plover is to interact with the outside world through STDIO
5. The protocol is based on line-separated JSON, with each message being on a separate line
6. Each message has a unique ID and replies must reference the ID
7. The unique ID for each request is generated by the client
8. The protocol must support these basic functionalities
   * Dictionary management: add, remove dictionaries, dictionary CRUD (entry update, deletion, addition)
